module Shrine::Plugins::RestoreCached

  1. lib/shrine/plugins/restore_cached.rb

The restore_cached plugin ensures the cached file data hasn't been tampered with, by restoring its metadata after assignment. The user can tamper with the cached file data by modifying the hidden field before submitting the form.

Firstly the assignment is terminated if the cached file doesn't exist, which can happen if the user changes the “id” or “storage” data. If the cached file exists, the metadata is reextracted from the original file and replaced with the potentially tampered with ones.
